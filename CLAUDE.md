## Where things live

- **All editable content** (brand, phone, services, portfolio, FAQs, blog posts,
  testimonials, `siteConfig.url`) → **`src/lib/site.ts`**.
- SEO helpers → `src/lib/seo.ts`; JSON-LD → `src/lib/schema.ts`.
- Layout/header/footer/mobile nav → `src/components/layout/`.
- Notebook UI (sticky notes, doodles, highlights, accordion) → `src/components/notebook/`.
- Homepage + page sections → `src/components/sections/`.
- Drawing layer → `src/components/drawing/`; smooth scroll + page transition →
  `src/components/providers/`.
- **OG/icon images are static PNGs** (`public/og.png`, `src/app/icon.png`,
  `src/app/apple-icon.png`) generated by `scripts/gen-images.mjs` (sharp,
  dev-only). Regenerate: `npm run gen:images`. Do **not** reintroduce `next/og`
  route files (see Gotchas).

## Deploy / update workflow (do this automatically on every change)

The user wants the **full pipeline handled without being asked**:

1. Make the edit(s).
2. **Verify a clean build:** stop any running node first, then
   `rm -rf .next && npm run build` (a stale dev server corrupts `.next`).
   Never push broken code — it's a live business site.
3. `git add -A && git commit` (end message with the Co-Authored-By line) then
   `git push origin main`.
4. Vercel **auto-deploys `main` to www.kvai.in** — no manual `vercel deploy`
   (a direct `vercel --prod` even fails on the folder name). Just confirm:
   `vercel ls` (newest Production = ● Ready) and `curl -I https://www.kvai.in`.
5. Report the live result.

Never create a duplicate Vercel project (local dir is already linked via
`.vercel/`). Never commit `.vercel/`, `.next/`, or `node_modules/`.

## Gotchas

- **Spaces in path:** `D:\KV Claude\Notbook` contains a space. `next/og`'s
  `ImageResponse` crashes at build (`fileURLToPath` on `import.meta.url`) because
  of it — that's why OG/icons are static PNGs, not `next/og` routes.
- **Mobile menu + backdrop-filter:** the scrolled header uses `backdrop-blur`,
  which makes it a containing block for `position: fixed` descendants. The mobile
  menu must stay **outside** `<header>` or it gets trapped in the header box and
  won't open after scrolling. (Already fixed — keep it that way.)
- **Windows builds:** kill stray `node.exe` before a clean build to avoid file
  locks; clean `.next` if dev and build ran against the same dir.
